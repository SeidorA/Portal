Crestone and Xtract Universal are solutions for data extraction from SAP to various platforms. 

Although both allow the transfer of information to different destinations, there are key differences in their approach, implementation and additional functionalities. 


The following comparison highlights the most relevant aspects to evaluate which one best suits the needs of the technical team


| Criterion            | Crestone                                                                 | Xtract Universal                                                                 |
|----------------------|--------------------------------------------------------------------------|----------------------------------------------------------------------------------|
| **SAP connection**   | ODATA, ODP(SAPI), TABLE                                                  | ODP, TABLE, CDC, BAPIS                                                           |
| **SAP Extraction**   | Data to multiple destinations (stage, table)                             | Data to multiple destinations (stage, table)                                     |
| **Workspace**        | Yes                                                                      | No                                                                               |
| **Alerts**           | Yes, in application and mail                                             | No                                                                               |
| **Supported destinations** | AWS S3, Snowflake, Azure, SQL server, Databricks, Google Cloud Storage y BigQuery | More variety (Azure SQL, Oracle, MySQL, SAP HANA, Power BI, Tableau, etc.)       |
| **Automatización**   | Contains basic and advanced scheduling functionalities integrated in the same platform | It has no scheduling functionality of its own. Must rely on third-party tools (Windows task scheduler) |
| **Implementation**   | Cloud & Self-hosted                                                      | Local installation                                                               |
| **User assistance**  | • Rapid deployment (SAP) <br /> • JOBS import and export                   | Allows cloning executions                                                        |
| **Licensing**        | No limit on number of destinations. Non-productive SAP env. allowed with same license | Each destination requires addon. Non-productive SAP requires more expensive license |
| **Orchestration**    | User-friendly and easy-to-use interface for load orchestration           | No load orchestration interface. Must be defined by command lines                 |
| **Support**          | Local support from brand owner                                           | Remote support through partners                                                  |




## Target Audience
Technical professionals with experience in SAP, ABAP and data lakes, looking for an efficient and cost-effective solution for data extraction.


## Key Use Cases

✓ Companies that need to extract and process SAP Z reports.

✓ Teams looking to automate data loading without relying on external command lines or configurations.

✓ Organizations that work with Databricks and require direct integration

## Advantages of Crestone

✓ Advanced Scheduled Execution: Allows scheduling jobs with multiple nodes directly in the platform through “JOBS”, without relying on command line or API.

✓ Z-Report support: Extracts custom SAP reports generated by users, something that Xtract Universal does not offer natively.

✓ Databricks integration: Allows data to be exported directly into tables within Databricks, optimizing data lake workloads.

✓ Improved experience: More intuitive and accessible interface for technical users
