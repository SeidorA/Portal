## 🚀 Nuevos orígenes, nuevo destino y más control sobre la integración de datos 

Seguimos evolucionando las capacidades de **integración de Crestone** para hacer más simple el acceso y la gestión de datos. 
<!-- truncate -->

Esta actualización permite extraer información desde distintas fuentes y llevarla a diversos destinos de forma más ágil. 

Además, incorpora **SQL Server** y **Azure SQL Server** como nuevas fuentes, **SAP HANA** como destino, un modo optimizado para consultas SQL personalizadas y mejoras en la configuración visual junto con el preview de datos, facilitando una experiencia más clara y flexible. 


![SQL Server y Azure SQL Server como fuentes de datos](https://supa.portal.seidoranalytics.com/storage/v1/object/public/News/campaigns/abril/abril_a.png)

## SQL Server y Azure SQL Server como fuentes de datos

Ahora Crestone permite configurar SQL Server y Azure SQL Server como fuentes de datos para extraer tablas y vistas, con soporte para cargas completas y, cuando aplique, cargas incrementales.

Se incorporó una experiencia visual de configuración para que el usuario pueda definir el origen directamente desde la app, navegando la estructura disponible y seleccionando:

- Base de datos
- Esquema
- Tabla
- Columnas

Además, se agregaron indicadores de carga para acompañar mejor el proceso mientras se consulta la información disponible desde la conexión.
                    
👉 En resumen: ahora SQL Server y Azure SQL Server pueden utilizarse como orígenes de datos dentro de Crestone, con una configuración visual guiada y más flexible, ampliando las opciones disponibles más allá de los orígenes SAP.

![SAP HANA como destino de extracción](https://supa.portal.seidoranalytics.com/storage/v1/object/public/News/campaigns/abril/abril_b.png)


## SAP HANA como destino

Crestone permite cargar datos extraídos en SAP HANA como destino final, con soporte para cargas completas y, cuando aplique, cargas incrementales.

Se mejoró la experiencia de configuración para que el usuario pueda definir con mayor claridad cómo se impacta la información en destino. Desde la interfaz, ahora es posible trabajar con esquemas y tablas de forma más ordenada: seleccionar un esquema existente, crear uno nuevo, buscar tablas disponibles o definir manualmente una nueva tabla de destino.

También se mantiene correctamente la configuración guardada al editar un destino existente, incluyendo esquema, tabla y la opción de eliminar datos existentes antes de insertar.

*👉 En resumen: SAP HANA puede utilizarse como destino final para los datos extraídos, con una configuración más clara, flexible y consistente*


## ✨ Experiencia en la app

![App Experience](https://supa.portal.seidoranalytics.com/storage/v1/object/public/News/campaigns/abril/abril_c.png)
                    


### Nuevo SQL Script Mode

Se incorporó un modo SQL Script para escribir consultas SQL personalizadas directamente desde la interfaz.

El usuario puede alternar entre el modo visual tradicional y el modo script. En el modo visual, puede seleccionar tablas y columnas desde la app. En el modo script, puede escribir la consulta completa en un editor SQL con formato, resaltado de sintaxis y ajuste de líneas.

*👉 En resumen: cuando necesitas una consulta más específica, puedes escribir el SQL completo y Crestone lo ejecuta tal como fue definido.*

---

### Preview para SQL Server y Azure SQL Server

Se agregó una vista previa de resultados para las nuevas fuentes SQL Server y Azure SQL Server.

Además, se mejoró el manejo de estados cuando falta seleccionar una tabla, campos o cuando no se encuentran datos, evitando mensajes confusos durante la configuración

*👉 En resumen: antes de avanzar con la configuración, puedes validar de forma más clara qué datos está devolviendo la fuente*

---

### Búsqueda de tablas más cómoda en ABAP y SAP S/4HANA

Se mejoró la experiencia de búsqueda de tablas en fuentes ABAP y SAP S/4HANA.

Ahora el campo queda libre para nuevas búsquedas luego de seleccionar una tabla, muestra estado de carga y ordena mejor los resultados, priorizando las tablas que empiezan con el texto buscado.

*👉 En resumen: antes de avanzar con la configuración, puedes validar de forma más clara qué datos está devolviendo la fuente*

---

### Mejor experiencia en configuración de BAPI

Se ajustó la configuración de fuentes BAPI para que sea más clara durante el armado del nodo.

Ahora se muestra mejor la BAPI seleccionada, se agregó la posibilidad de limpiar la selección, se diferencian visualmente los campos sin valor y se mejoró el estado de carga mientras se obtiene la metadata.

*👉 En resumen: configurar BAPIs ahora es más claro y menos propenso a confusión.*


---

### Paginación más ordenada

Se ajustó la paginación en pantallas con muchos nodos.

Cuando hay muchas páginas disponibles, Crestone ahora muestra una navegación resumida con puntos suspensivos, evitando barras largas y difíciles de leer.

*👉 En resumen: navegar listados grandes ahora es más limpio y cómodo.*
